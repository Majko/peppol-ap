Ready-to-use drafts for the OpenPeppol membership application, the PA SK introductory email, and a full certification checklist you can track.

1. PA SK Introductory Email

Send this to peppol@financnasprava.sk as the first contact. It expresses your intent to begin accreditation and asks if any additional requirements apply beyond the published documents. This also puts you on their radar.

📧 Email to: peppol@financnasprava.sk

Subject: Accreditation of a new Service Provider – [Company name], IČO: [IČO]

Dear Peppol Authority of the Slovak Republic,

we would like to inform you that [Company name], IČO: [IČO], with registered office at [address], intends to begin the accreditation process as a Service Provider (Access Point) in the Peppol network in the Slovak Republic.

We have reviewed the published documents:
- Accreditation Scheme of PA SK (April 2026)
- Detailed instructions for accrediting service providers (April 2026)
- Peppol Authority Specific Requirements for the Slovak Republic

We are currently preparing the required documentation and will submit the application via the Finančná správa portal shortly.

Could you please confirm whether any additional requirements or country-specific conditions apply beyond what is stated in the published documents? We would also appreciate guidance on the current status of PASR points 4 (Centralised Slovak SMP) and 6 (Slovak TDD) — are these mandatory for new applicants as of July 2026?

Please find our company details below:

  Legal name:    [Company name]
  Legal form:    [s.r.o. / a.s. / živnosť]
  IČO:           [IČO]
  DIČ:           [DIČ]
  IČ DPH:        [IČ DPH, ak existuje]
  Registered in: Obchodný register [Okresný súd], oddiel [X], vložka č. [YYYY]
  Contact email: [email]
  Contact phone: [phone]

We look forward to cooperating with you.

Sincerely,

[Your name]
[Title, e.g. konateľ]
[Company name]
💡 Tip: If you already have a specific question about the process or timeline, include it. The PA SK team has been responsive to applicants based on community reports. Send this email first, then proceed with the formal application — the 30-day evaluation clock starts when the formal application is submitted, not when this email is sent.

These are the code changes needed before we can pass the Peppol Testbed. Estimated effort: 2–4 weeks.

Done#ItemPriorityDetails
☐T1Remove dryrun: true hardcodeCriticalsrc/as4/node42.js — controlled by AP_CORE_DRY_RUN env var, default false. Already planned in Slice 3.
☐T2Wire certificate loading from identity storeCriticalRemove filesystem cert loading. Use APIdentityStore.getActiveCert(). Add cert expiry validation before send. Slice 3.

☐T6Implement proper AS4 error responsesHighTestbed expects specific error codes for: invalid participant (EB:001), malformed message (EB:002), expired cert, processing error. Map all error scenarios to correct AS4 error codes.
☐T7Add SMP cache with persistenceHighFor Testbed performance: cache participant lookups so repeated lookups of the same ID don't hit DNS/SMP every time. Slice 2 covers this.
☐T8Set up persistent transaction storeHighTestbed tests may run over multiple sessions. Transaction log must survive restarts for duplicate detection. Slice 2 (SQLite) covers this.

8. Key Lessons Learned (from community sources)

  • Start the Testbed early. Don't wait for the formal evaluation to finish. You can request the test PKI cert as soon as both applications are submitted (you don't need the evaluation to complete first).
  • The 30-day evaluation clock can pause. If PA SK asks for supplements, the clock resets. Submit your application with meticulous documentation to avoid this.
  • PA SK is responsive. Community reports indicate the Slovak authority replies to emails within 2–3 working days. Use this to your advantage — ask questions early.
  • The Testbed is self-service. You don't need to schedule anything with OpenPeppol for the eDelivery tests. You log in, run tests, get results. The Slovakia-specific tests may require coordination with PA SK.
  • G3 PKI migration is done. As of April 2026, all certificates are on the new DOTL CA chain. Node42 already supports this. Just make sure you request G3 test certificates specifically.
  • Don't underestimate the Slovak-specific testing. Several community posts mention that the Slovakia Testbed Report (national extensions, IS EFA format) is where most applicants hit delays. Budget extra time for this.
