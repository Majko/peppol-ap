# Established prior knowledge: full-stack Node.js proficiency

The user knows Node.js, Express, PostgreSQL, Vite, and React well — no need to teach the basics of web frameworks, database design, REST APIs, or frontend development. This is already fluent knowledge.

## Implications

- Lessons can focus entirely on the Peppol domain layer — the transport protocols (AS4), document formats (UBL XML), PKI certificate management, and the Peppol-specific business logic.
- Architectural discussions can assume comfort with patterns like message queues, async processing, relational data modelling, and API design.
- The hard part is not "how do I build a server" but "what does this server need to do to be a Peppol-compliant Access Point."
